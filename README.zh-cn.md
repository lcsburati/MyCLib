## 目的

**目的：** 该项目旨在通过模拟一些库来理解C语言的行为，基于我对它们应该如何工作的理解，而不一定是它们实际的工作方式。目标是使用我目前的知识达到类似的结果，专注于理解语言语法、其原始和派生数据结构等方面。目的是真正理解低级语言的工作原理，理解幕后发生的抽象。此外，我想加深我的知识并练习逻辑推理，以改进我解决问题的方式。

## 如何编译 (CMake)

您可以使用 **CMake** 编译项目：

```bash
mkdir build && cd build
cmake ..
make
./main
```

## 函数

### string_utils.h

> 这里的目标是存储原型、结构和常量，以保持 source string_utils.c 更简洁。

* `#define OFFSET ('a' - 'A')`：定义小写和大写之间距离的常量（`32`）。用于 my_tolower 和 my_toupper。

* `#define STR_BUFFER 256`：定义字符串可以拥有的最大字符大小。

* 注意：选择 #define 而不是 const 是由于变量的简单性，因为在运行时类型不会很重要。

* `typedef char * string`：为 char* 定义别名。我们可以直接使用指针工作，但为了便于理解字符串在底层的工作原理，我选择使用 string 作为 char* 的别名。

### string_utils.c

> 这是 string_utils.h 的配对文件，是应用头文件中定义的原型函数逻辑的地方。

<details>
<summary><strong>my_strlen</strong> (点击展开)</summary>

在这个函数中，输入参数是 `string` 类型（`char*` — 指向 `char` 的指针）。  
函数的返回类型是 `size_t`，选择它是为了方便与 unsigned int 的大小相关的可移植性，因为我们处理的是数组（字符串）的大小。

我们首先定义将计算字符串（数组）长度的变量，这里命名为 `length`，也是 `size_t` 类型，原因与函数返回值相同（另外，它恰好是返回的值）。

从那里开始的逻辑很简单：知道C中的每个字符串都以空字符 `\0` 结尾，我们可以遍历字符串直到找到它。这个字符被解释为 `0` 或 `false`（因为布尔值在C中不是原生存在的），所以我们可以简单地使用 `while (*str)` 来指示程序它应该遍历从输入字符串开始的指针，直到找到 `\0`。每次循环迭代，我们将 `length` 增加1，累积字符串大小。

**注意：** 我们在 `while` 条件中添加了 `length < STR_BUFFER` 检查，以确保输入受到保护，防止非常长的字符串，避免潜在的 _overflows_ 或不希望的等待时间。此外，这是良好的安全实践。

**注意2：** 函数也将空格（` `）识别为一个字符。也就是说，如果您传递带有空格的字符串，返回的大小将是字符串的总大小，包括空格。我认为重要的是要提到这一点，因为根据上下文，一些函数/方法将空格视为分隔符，最终会"忽略"这个字符（例如，使用 `scanf("%s")` 读取输入时）。

<summary><strong>my_toupper</strong> (点击展开)</summary>

在这个函数中，输入参数是一个字符串（char指针），命名为 `str`。

我们首先定义处理所需的变量：`length`（`size_t`）和 `result`。对于 `result`，我们通过 `malloc` 函数在*堆*上分配内存，创建一个指针，标记一个块的开始，块的大小正好是输入字符串的大小 + 1字节空间用于空字符 `\0`。

分配后，我们验证 `malloc` 是否成功；否则，函数返回 `NULL`（防止如果内存中没有连续空间进行块分配时出现故障）。

我们还定义了一个辅助指针 `string current = result`。这允许我们遍历和填充新的内存块，而不丢失对块开头（`result`）的引用，这对于函数返回是必要的。

转换逻辑遍历输入字符串（`str`）：我们检查当前字符是否是小写字母；如果是，我们通过从字符的 ASCII 值减去 `OFFSET` 值（32）将其转换为大写。否则，我们只将原始字符复制到新块中。我们还在条件内检查缓冲区以确保操作安全。

在循环结束时，我们将 `current` 的最后一个字符设置为 `\0`，正确终止字符串。我们返回 `result`，它指向现在处于 **UPPERCASE** 的新字符串的开头。

</details>

<details>
<summary><strong>my_tolower</strong> (点击展开)</summary>

在这个函数中，我们遵循与 `my_toupper` 相同的内存管理逻辑，使用输入指针 `str` 并使用 `malloc` 在*堆*上分配一个新块来存储结果。

根本区别在于转换逻辑：我们遍历输入字符串，检查当前字符是否是大写字母。如果是，我们通过将 `OFFSET` 值（32）添加到字符的 ASCII 值来将其转换为**小写**。

与其他函数一样，我们通过检查内存分配的成功与否并控制缓冲区限制来保持安全。在最后，我们确保字符串以空字符 `\0` 终止，并返回 `result` 指针，它指向转换为 **lowercase** 的新字符串的开头。

</details>

<details>
<summary><strong>my_strcmp</strong> (点击展开)</summary>

输入参数是两个要比较的字符串，返回值是一个根据比较结果变化的 `int`：如果不同且第一个字符串较大，返回1；如果不同且第一个字符串较小，返回-1；如果相等，返回0。

这里的目标是比较两个字符串并返回比较结果。

我们首先定义两个将指示两个字符串大小的变量。

从这里，我们做一个 `if` 来检查哪个字符串更大。如果是第一个字符串，我们用 `while` 遍历它（因为我们使用指针工作）。在这个 `while` 中，我们已经添加了缓冲区检查，并在其中比较 `*str1 != *str2`。如果为真，返回1；否则，我们遍历整个字符串，如果它们相等，条件不满足，我们返回0。

如果第二个字符串更大，相同的逻辑应用于 `else if`，返回-1。

现在，如果它们大小相同，我们使用 str1 遍历字符串并与 str2 比较。如果不同，返回1；否则，我们遍历整个字符串，如果它们相等，条件不满足，我们返回0。

</details>

<details>
<summary><strong>my_strcmp_percent</strong> (点击展开)</summary>

输入参数是 `str1` 和 `str2`，类似于 `my_strcmp`。主要区别在于算法：要计算相似度百分比，我们使用**莱文斯坦距离**，它根据将 `str2` 转换为 `str1` 所需的最少操作数填充矩阵——这些操作是：字符的**插入**、**替换**和**删除**。

算法首先定义 `str1` 和 `str2` 的大小，然后在堆上分配 `matrix`——作为双指针（`int**`）处理——使用 for 循环通过 `malloc` 为行和列分配空间。

分配后的前两个循环定义**矩阵边框**：左边框（列0）和上边框（行0）填充递增的值，表示从零开始的成本——也就是说，如果其中一个字符串为空，需要多少操作。

在主循环（遍历行和列的两个嵌套循环）中，我们通过三元条件定义 `cost` 变量：如果比较的字符相等，`cost = 0`；否则，`cost = 1`。

然后，我们取**3个操作中的最小值**——因为我们想要到达结果的最小成本路径，因为可以用不同的方式将一个字符串转换为另一个字符串，但成本不同。每个操作由当前单元格的3个相邻单元之一表示：

- **`[i-1][j]` → 删除：** 我们只在一个方向（str1）前进，意思是那个字符应该被忽略——成本+1。
- **`[i][j-1]` → 插入：** 我们只在 str2 的方向前进，也就是说，我们比较的字符串缺少1个字符——成本+1。
- **`[i-1][j-1]` → 替换（或匹配）：** 我们在两个方向都后退一个，保持同步——如果字符不同成本+1，或者如果相同成本为0。

循环完成后，矩阵被完全填充，`matrix[str1_length][str2_length]` 中的值是所有操作的最终累积距离。

从这里，我们使用三元条件来识别哪个字符串更大，并使用其大小作为百分比计算的分母。最后，我们释放分配的矩阵空间并以 `double` 形式返回相似度。

</details>
